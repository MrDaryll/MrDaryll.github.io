博客简易搭建
发布于 2019-12-17   更新于 2019-12-17

一： 准备工作
· 拥有一个github账号，没有的话可以去注册一个：https://github.com
· 安装node.js可自行下载： https://nodejs.org/zh-cn/download/
· 安装git for windows（或者其它git客户端）: https://git-scm.com/downloads/
· 网盘（git+node.js安装包）链接： https://pan.baidu.com/s/18XnUFJ108uKe283Zbk7Zsw 提取码： j15y 

本次搭建所使用的环境：
    · Windows7
    · node.js -v12.10.0
    · git -v2.23.0
    · hexo -v3.9.0
文本编译：
    Notepad++
（文中 “##” 表注释，可忽略）

二：搭建Github
新建一个《用户名.github.io》的Lirbrary，比如说，如果你的github用户名是Blog，那么你就新建Blog.github.io的Library（用户名必须同你的github用户名一样，其它名称无效），将来你的网站访问地址就是 https://Blog.github.io 了。

几个注意的地方：
    github注册的邮箱需要验证，否则不会成功;
    仓库名字必须是：username.github.io，其中username是你github的用户名;

    创建成功后，默认会在你这个仓库里生成一些示例页面-可忽略.

三：配置SSH key
（之前已安装好git，在C:盘或者D:盘-根目录新建一个文件夹进入，鼠标右击选择Git Bash Here进行编辑，此处随个人喜好）

输入：（##为注释）
    $ cd  ~/.ssh  ## 检查本机是否已经存在ssh秘钥

如果提示：
    No such file or directory  说明你是第一次使用git。

输入：
    $ ssh-keygen -t rsa -C “邮箱地址”

为了访问方便，建议一路Enter,最后得到信息：
    Your public key has been saved in /c/Users/user/.ssh/id_rsa.pub.
    最终会生成一个文件在C:\Users用户目录下，打开用户目录，找到.ssh\id_rsa.pub文件，记事本或者其他编译器打开并复制里面的内容，打开你的github主页，进入设置settings -> SSH and GPG keys ->(SSH keys) New SSH key，将刚复制的内容粘贴到key那里，title可随便填(也可不填)，保存即可。

测试是否添加成功：
    输入：
        $ ssh -T git@github.com  ## 注意邮箱地址不要写错

    如果提示：
        Are you sure you want to continue connecting (yes/no)?

    输入: yes 然后会看到：
        Hi liuxianan! You've successfully authenticated, but GitHub does not provide shell access.

    看到这个信息说明SSH已配置成功！
    此时你还需要进行配置：
    输入：
        $ git config --global user.name “github用户名”  ## 填写你github用户名
        $ git config --global user.email “github邮箱地址”  ## 填写你github注册的邮箱

四：安装Npm、Hexo
官网：https://hexo.io

初始化：
    可新建一个文件夹（名字自己随便取），比如我的是D：\GitLibrary，由于这个文件夹将来就作为存放代码的地方，所以最好不要随便放。
    （例： 进入D:盘新建的文件夹GitLibrary，鼠标右击选择Git Bash Here进行编译）
安装 Npm:
    输入：
        $ npm install
        $ npm install -g hexo-cli  
安装Hexo:
    输入：
        $ hexo init
    成功会出现提示如下！
        INFO Start blogging with Hexo!

    执行完hexo init会自动下载一些文件到该目录下。
初始化hexo后，在该目录下source目录会自生成一个hello-world.md的文件，可直接执行以下命令启动服务：
    输入：
        $ hexo g  ##生成（hexo generate简写）
        $ hexo s  ##启动服务（hexo server简写）

    出现提示：
        INFO  Start processing
        INFO  Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.

    打开浏览器访问 http://localhost:4000 即可看到一个精简版的网页。

    主题修改，可访问官网：
        官网： https://hexo.io/themes/

五：部署Hexo
先配置根目录下_config.yml中相关的部分（使用编译器打开）：

## Site
    title:  标题 
    subtitle:  副标题
    description: 描述
    author: 名称
    language: zh-ch 
    timezone: Asia/Shanghai 
配置deploy部署（repo处url修改成您自己的github-urll地址）
    deploy: 
       type: git  
       repo: https://github.com/scrupulousLy/scrupulousLy.github.io.git   
       branch: master  

注：编辑时切记在( : ) 后添加一个空格，完成后保存。
    （如果遇到无法上传可将该处修改为 repo：git@github.com:scrupulousLy/scrupulousLy.github.io.git）

六：发表博客
执行hexo 命令：
$ hexo new 测试文章
# 注: 如果标题存在空格需要加上双引号” “.
出现：
INFO Created: D:\GitLibrary\source_posts\测试文章.md
根据路径可找到该文件，打开编译器可进行编译。
保存后执行下列hexo命令：
$ hexo clean ##清楚缓存
$ hexo g ##生成
$ hexo s ##预览 

此时，打开浏览器访问 http://localhost:4000，你会发现刚才的文章已经发布成功了。
执行博客部署。
    $ hexo d ##进行部署 
执行时会出现一个错误提示。
    Deployer not found：github  或者 Deployer not found： git
这是缺少一个插件：
    输入：
        $ npm install hexo-deployer-git --save
安装完，再次执行hexo d 部署成功。
